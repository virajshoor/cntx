Quick start

Install and open the terminal docs.

Cntx ships public docs in the crate and embeds the most important pages into the binary. That means a user can run the docs browser even after installing from crates.io or Homebrew.

First run

Configure one endpoint, then refresh models.

Use cntx init for a guided setup, or pass flags for a non-interactive setup. Keys go into the runtime secrets store or an environment variable, not into source.

Apply mode

Write model-proposed files through the sandbox.

In apply mode, Cntx asks the model for complete fenced code blocks annotated with path=, file=, or filename=. It previews those blocks, writes only path-annotated files through the edit sandbox, and prints a checklist. 

```rust path=src/example.rs
pub fn example() {}
```

Providers

Use Anthropic, OpenAI, Ollama, or OpenAI-compatible gateways.

Cntx supports first-class provider endpoints and reusable provider presets. The built-in gallery includes OpenRouter, Groq, Together, and Fireworks.

Safety

The edit sandbox is on by default.

File writes are confined to the project root unless you explicitly add another writable directory. Even in allow mode, outside paths are denied unless the sandbox is widened.

Project memory

Keep lightweight repo preferences in .cntx/memory.md.

Memory is intentionally plain markdown so it can be reviewed, edited, committed, or ignored like any other project note.

Sessions

Resume the latest session or export one for tooling.

Sessions are stored as YAML in the config directory. Use the latest session with no id, or export a specific session to JSON.
